# AuraClaw 审核层

这个目录用于保存人工审核状态和审核历史。

## 使用方式

- 管理后台登录页：`/admin/login`
- 管理后台审核页：`/admin/review`
- 普通前台不会展示审核入口
- 只有管理员登录后，才可以查看和修改这些审核状态

## 设计原则

- 原始采集文件永远保留在 `content/inbox/source-harvest/`
- 审核状态单独保存，不覆盖采集结果
- 所有审核动作都保留历史，支持回溯
- 原始采集结果和审核结果分离，避免误删和误覆盖

## 文件说明

- `state.json`
  当前每条候选内容的最新审核状态

- `history.jsonl`
  审核动作历史日志，一行一条记录

- `drafts/source-notes/`
  批准为来源索引后自动生成的草稿文件

- `drafts/recipes/`
  批准为经验包后自动生成的草稿文件

- `snapshots/`
  草稿被再次生成前保留的历史快照，支持回溯

## 审核状态

- `pending`
- `approved_source_note`
- `approved_recipe`
- `rejected`
- `archived`

## 审核后会发生什么

- 批准为来源索引：自动生成一份来源索引草稿
- 批准为经验包：自动生成一份经验包草稿
- 重新批准同一条内容：会覆盖当前草稿，但旧版本会进入 `snapshots/`
- 拒绝或归档：不会删除已经生成过的草稿，方便后续回看

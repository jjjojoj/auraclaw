# AuraClaw 内容来源模板与引用规则

更新时间：2026-03-22

## 1. 这条线为什么要先定规则

AuraClaw 后面一定会持续从外部站点吸收内容，但不能把网站做成“内容搬运站”。

更合适的方式是：

- 先设计统一模板
- 再抓取公开信息
- 只收录对 AuraClaw 有价值的结构化内容
- 每条都保留来源和引用方式

这样做的好处：

- 首页不会重新变成信息垃圾场
- 后续内容能稳定落入同一套结构
- 用户看到的是被整理过的结果，不是原文堆砌
- 站点也更容易长期维护

## 3. AuraClaw 里真正要收录的内容单位

不是整篇文章，也不是整个网页。

AuraClaw 更适合收录的是下面这些内容单位：

- 一个清晰的教程条目
- 一个高价值问题的答案
- 一个能力说明
- 一个场景案例
- 一个安装准备清单
- 一个适合引用的路径导航

也就是说，采进来的不是“网页”，而是“内容颗粒”。

## 4. 每条采集内容统一要有的字段

建议每条内容都按下面结构落库：

- 标题
- 来源站点
- 原始链接
- 内容类型
- 适合放入的板块
- 一句话摘要
- 适合哪类用户
- 是否适合新手
- 是否需要人工复核
- 可引用方式
- 最后检查时间

## 5. 可引用方式建议

建议分成 3 档：

### A. 只做链接引用

适合：

- 教程目录
- 页面导航
- 学习路径

### B. 做摘要引用

适合：

- 教程章节
- FAQ
- 案例介绍

做法：

- 提炼一句摘要
- 给原始链接
- 标明来源站点

### C. 做结构化重写

适合：

- 能力说明
- 入门步骤
- 开始前准备
- 场景案例模板

做法：

- 保留来源
- 改写成 AuraClaw 自己的统一模板
- 不直接搬运原网页整段正文

## 6. 前端页面应该怎么接这些内容

首页不要直接接爬来的内容。

首页只负责：

- 让用户知道自己该从哪里开始
- 展示少量精选样板间
- 把用户导向板块页或详情页

真正适合承接爬取内容的位置是：

- 板块页
- 经验详情页
- 来源索引页

## 7. 推荐的内容模板层级

建议后面按 3 层来做：

### 第一层：样板间

面向首页展示。

特点：

- 文案短
- 气质强
- 不解释太多

### 第二层：经验说明

面向用户使用。

特点：

- 讲清楚这条经验能帮什么
- 怎么开始
- 怎么验证
- 失败怎么办

### 第三层：来源条目

面向内容沉淀和引用。

特点：

- 记录来源
- 记录摘要
- 记录适合放到哪里
- 支持长期复用

## 8. 当前建议的执行顺序

建议这样推进：

1. 先把首页和板块模板收住
2. 再定“来源条目”模板
3. 先手动收录 20 到 30 条高价值内容颗粒
4. 再决定是否做半自动采集

不要一开始就大规模抓正文，否则很容易把 AuraClaw 再做回内容堆。

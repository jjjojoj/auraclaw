# AuraClaw 内容来源模板与引用规则

更新时间：2026-04-08

## 1. 这条线为什么现在必须收住

AuraClaw 接下来会越来越依赖外部生态里的真实经验，但不能把网站做成“教程搬运站”或者“信息垃圾堆”。

更稳的方式是：

- 先定来源矩阵
- 再定可复用的内容颗粒
- 最后把内容改写成 AuraClaw 自己的经验包

也就是说：

- 我们参考公开站点
- 我们引用公开来源
- 但我们最终交付的是 AuraClaw 自己的结构化经验

## 2. 这轮优先参考的来源矩阵

### [OpenClaw101.club](https://www.openclaw101.club/)

适合拿来做什么：

- 新手路径分流
- 中文教程目录
- 渠道接入导航
- 进阶功能导航
- 案例题目和 FAQ 方向

当前观察到的强点：

- 首页就把人群分成小白、开发者、企业 / 团队
- 明确列出快速入门、渠道接入、技能系统、浏览器自动化、多 Agent 架构等主题
- 有“企业适合引入 OpenClaw 吗”这种决策型内容

最适合放进 AuraClaw 的位置：

- `产后护理`
- `能力扩展`
- `对话训练`
- 企业向的 `一人公司（OPC）` 前置判断

### [Claw101](https://claw101.com/)

适合拿来做什么：

- 章节化学习路径
- 开始前准备清单
- 可落地案例标题
- 最新实战趋势

当前观察到的强点：

- 强调“Not demos — real deliverables”
- 免费教程有 13 个章节，从准备工作到技能系统、浏览器控制、自动化
- Blog 和 Cases 在持续更新真实部署与多实例场景

最适合放进 AuraClaw 的位置：

- `产后护理` 的起步包
- `能力扩展` 的前置准备
- `对话训练` 的真实任务表达样例
- `一人公司（OPC）` 的工作流灵感

### [openclaw101 GitHub 仓库](https://github.com/mengjian-github/openclaw101)

适合拿来做什么：

- 来源结构地图
- 中文来源清单
- 7 天学习路径骨架
- 平台接入与长期成长的路线排序

当前观察到的强点：

- `src/data/resources.ts` 本质上是一份 OpenClaw 来源数据库，适合反哺 AuraClaw 的来源索引
- `content/days` 是一条很清楚的学习路径，尤其适合拿 day1、day4、day7 来强化我们的起步、能力扩展和 OPC 叙事
- 仓库里已经聚合了很多中文部署、平台接入和教程链接，适合转成 AuraClaw 的 source note

最适合放进 AuraClaw 的位置：

- `产后护理` 的起步心智和第一次成功路径
- `能力扩展` 的“先接入、再测试、再上真实任务”顺序
- `飞书 / 钉钉 / 企微接入后的第一条工作流`
- `一人公司（OPC）` 的长期成长路线

当前使用原则：

- 不搬它的整篇正文
- 不复刻它的人格化教程表达
- 重点拿它的来源分类法、中文来源条目和学习路径顺序
- 最终都要改写成 AuraClaw 自己的经验包或 source note

## 3. AuraClaw 最适合采集的内容单位

AuraClaw 不适合直接采一整篇文章。

更适合沉淀的是这些内容颗粒：

- 一个清晰的入门步骤
- 一个高价值的安装准备清单
- 一个能力说明
- 一个真实场景标题
- 一个可执行工作流的骨架
- 一个高频问答
- 一个值得继续追踪的技能方向

换句话说：

我们采进来的不是“网页”，而是“经验线索”。

## 4. 什么内容最适合放进 AuraClaw

### 适合直接改写成经验包的

- 安装准备
- 能力接入
- 渠道接入
- 浏览器控制
- 定时任务
- 知识库整理
- GitHub / Calendar / Notes 这类高频技能方向

### 适合做摘要引用的

- 案例文章
- 社区案例页
- FAQ
- 教程目录

### 只适合做来源链接的

- 首页导航
- 社群招募页
- 整站目录页

## 5. 绝对不要做的事

- 不要整页整页搬正文
- 不要让首页直接堆外部信息
- 不要为了“内容多”而牺牲 AuraClaw 的结构统一
- 不要把 skill 市场直接当成 AuraClaw 本身

AuraClaw 的价值不是“你去这里看看”，而是：

> 我们已经替你把这条路跑了一遍，并整理成了可以直接交给 OpenClaw 的经验包。

## 6. 推荐的内容模板层级

建议继续按 3 层走：

### 第一层：首页精选

只放少量强结果感内容。

### 第二层：经验包

这是用户真正会复制给 OpenClaw 的东西。

### 第三层：来源条目

只负责记录出处、摘要、适合改成什么。

## 7. 首批最值得做深的内容方向

基于这轮参考站点，我建议首批优先做这 12 条：

### 产后护理

- 30 分钟起步包：安装、onboard、第一条可执行任务
- 飞书 / 钉钉 / 企微接入后的第一条工作流
- 会议纪要 + 后续待办整理

### 能力扩展

- 浏览器控制与抓取增强
- GitHub 自动化能力接入
- 语音转文字 + 笔记入库

### 对话训练

- 定时任务表达校准
- 行业情报收集表达校准
- 多渠道发布任务表达校准

### 一人公司（OPC）

- 一人内容公司
- 一人行业情报台
- 一人知识库运营台

## 8. 当前建议的执行顺序

建议接下来按这个顺序做：

1. 先用这些站点做“来源条目”沉淀
2. 再从里面挑出首批 10 到 12 条经验包
3. 先把 `产后护理` 和 `能力扩展` 做深
4. 再补 `对话训练`
5. 最后再把多个经验包串成 `一人公司（OPC）`

这样推进，内容会越来越像 AuraClaw，而不是越来越像别人的资料站。

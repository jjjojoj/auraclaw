## 1. 这份文档的作用

这份文档只负责 3 件事：

- 告诉你现在哪份文档才是主线
- 告诉你旧文档被放到了哪里
- 告诉你当前原型和目录分别属于什么层级

一句话原则：

> 以后 AuraClaw 只按一份主控文档开发，其余文档全部归为历史或参考。

## 4. 当前唯一主线怎么用

以后如果有新想法，按下面顺序走：

1. 先补到 `docs/AuraClaw_主控文档.md`
2. 再决定要不要改页面
3. 页面改完后，再回写版本索引里的状态说明

以后不要再出现这种情况：

- 一份桌面文档在更新
- 一份 docs 文档在更新
- 一份页面代码又在单独长

这会很快把项目带乱。
